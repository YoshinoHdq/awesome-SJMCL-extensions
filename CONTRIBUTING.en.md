# Contributing Guide

Languages: English · [简体中文](CONTRIBUTING.md) · [繁體中文](CONTRIBUTING.zh-Hant.md)

Thank you for contributing to awesome-SJMCL-extensions!

This repository does not host plugin source code. We only collect links to plugin source repositories.

## Contents

- [Plugin Repository Requirements](#plugin-repository-requirements)
- [Submission Flow](#submission-flow)

## Plugin Repository Requirements

Before submitting to this list, your plugin repository must include:

- `README.md` — plugin introduction, feature overview, and usage instructions
- Accessible source code, a packaged plugin release, or a download link

These are the only hard requirements. The exact format and structure are up to the plugin author.

## Submission Flow

1. Fork this repository and create a new branch.
2. Add one entry to the corresponding category in `README.md`, `README.en.md`, and `README.zh-Hans.md`.

    ```markdown
    - **[Plugin Name](https://github.com/your/repo)** — One-line description.
    ```

    Example:

    ```markdown
    - **[Clock](https://github.com/example/sjmcl-clock)** — Displays a live clock in the launcher interface.
    ```

3. Open a Pull Request with a brief title describing the plugin you added.

> No other files need to be added to this repository.

## Acceptance Criteria

- The plugin is clearly related to the SJMCL ecosystem
- The plugin source repository is public and contains a `README.md`
- It does not violate any relevant open-source licenses
- It complies with the laws and regulations of the relevant region
- It is not a duplicate of an existing entry

## Submission Checklist

- [ ] Added the entry to all three language versions of the README
- [ ] The entry link points to the plugin source repository
- [ ] The plugin source repository contains a `README.md`
- [ ] The description is concise (one sentence)

If you have any questions, please open an issue.
